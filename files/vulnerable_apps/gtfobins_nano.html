Privilege Escalation — Sudo

If /usr/bin/nano can be run via sudo on a target file, you can escape to a root shell.

sudo nano /etc/hosts

Once nano opens:

  1. Press Ctrl-R then Ctrl-X (the read-file shortcut exits nano)
  2. When prompted for a file name, type: /bin/sh
  3. You now have an interactive root shell.

Note for this lab

Your sudo rule is: appuser ALL=(ALL) NOPASSWD: /usr/bin/nano /etc/hosts

Try running it and then apply the escape above.
